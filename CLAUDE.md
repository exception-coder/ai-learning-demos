# AI Learning Demos 项目开发指南（Claude Code）

## 核心规则（必须遵守，按优先级排序）

---

### 规则 1：知识点文档归档规则（最高优先级）

每次对话中出现新的技术知识点时，必须执行以下流程：

**Step 1 — 读取大纲**
读取 `docs/guides/README.md`，获取当前所有文档类目及其内容概要。

**Step 2 — 判断归属**

| 情况 | 处理方式 |
|------|----------|
| 知识点属于已有类目（单文档） | 追加到对应文档末尾合适章节，并更新 `README.md` 中该文档的内容概要 |
| 知识点属于已有组件目录 | 追加到该组件目录下对应文档，并更新组件目录的 `README.md` |
| 知识点不属于任何已有类目，且该知识点是**独立组件/框架**（本身涉及系列子知识） | 在 `docs/guides/` 下新建**组件目录**，目录内建 `README.md` 作为该组件的子大纲 |
| 知识点不属于任何已有类目，且是**单一概念** | 在 `docs/guides/` 下新建单个文档，并在顶层 `README.md` 中登记 |

**判断是否需要建目录的标准：**
- 该知识点是一个独立的组件、框架或工具（如 Superpowers、LangChain、LangGraph）
- 它本身包含多个子概念（安装、核心技能、使用场景、进阶用法等）
- 未来很可能继续补充相关知识

**组件目录结构：**
```
docs/guides/<component-name>/
├── README.md          # 该组件的子大纲索引
├── 01-overview.md     # 概述
├── 02-usage.md        # 使用方式
└── ...                # 其他子文档
```

**组件目录在顶层 `README.md` 的登记格式：**
```markdown
### XX. 组件名称（目录）
**目录**: `<component-name>/`

**内容概要**:
- 子文档1
- 子文档2

**适合人群**: ...
```

**单文档命名规范：**
- 使用下一个可用数字前缀：`08-`, `09-`, `10-`...
- 使用小写字母和连字符：`08-xxx-xxx.md`
- 文件名清晰表达文档主题

**禁止：**
- 不得跳过读取 `README.md` 大纲的步骤直接写文档
- 不得在 `docs/guides/` 以外的地方创建技术指南文档
- 不得在 `README.md` 中遗漏登记新文档

---

### 规则 2：学习日志自动记录

用户提出技术疑问时，**自动**追加到 `docs/learning-journal.md`，无需询问。

记录格式：
```markdown
### Q: 问题标题

**疑问：**
> 用户的原始问题

**解答：**
详细解答...

**关键要点：**
- 要点1

**常见误区：**
- ❌ 错误理解
- ✅ 正确理解
```

---

### 规则 3：开发日志记录

每次代码变更必须追加到 `docs/dev-log.md`：
- 日期
- 任务描述
- 创建或修改的文件
- 关键设计决策

---

### 规则 4：文件归属规则

所有新增文件必须写入当前项目，禁止写入全局目录。

| 场景 | 正确位置 | 错误位置 |
|------|---------|----------|
| 技术文档 | `docs/guides/` | 其他项目 docs |
| 代码文件 | 当前项目 `src/` | 其他项目 |
| Skill 文件 | `<当前项目>/.cursor/skills/` | `~/.cursor/skills/` |

---

## 任务完成检查清单

- [ ] 涉及新知识点 → 已读取 `docs/guides/README.md` → 已归档到对应或新建文档
- [ ] 技术疑问 → 已追加到 `docs/learning-journal.md`
- [ ] 代码变更 → 已追加到 `docs/dev-log.md`
- [ ] 新增文件在当前项目内

## 文档结构说明

```
docs/
├── guides/
│   ├── README.md                  # 顶层大纲索引（每次新增/修改后必须同步）
│   ├── 01-xxx.md                  # 单一概念文档
│   ├── <component-name>/          # 组件目录（涉及系列知识的独立组件）
│   │   ├── README.md              # 该组件的子大纲索引
│   │   ├── 01-overview.md
│   │   └── ...
│   └── ...
├── learning-journal.md            # 学习疑问与解答记录
├── dev-log.md                     # 开发变更日志
├── learning-notes.md              # 学习笔记
└── quick-start.md                 # 快速开始
```
